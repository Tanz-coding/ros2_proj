# 课程项目架构说明（参考概要设计）

## 1. 总体架构（分层）

本项目采用“仿真层 → 基础能力层 → 任务层 → 数据记录层”的四层结构。

- 仿真层：Gazebo + TurtleBot3，负责提供可运行机器人与传感器数据。
- 基础能力层：SLAM / 地图服务 / Nav2，负责定位、建图、路径规划与控制。
- 任务层：多站点投递、让行、到站拍照等业务逻辑。
- 数据记录层：CSV 日志与图片结果，支撑验收、报告与演示。

## 2. 模块划分

### 2.1 仿真与地图

- Gazebo 仿真入口（外部标准包）：`turtlebot3_gazebo`
- 自主探索：`autonomous_tb/exploration_node.py`
- 地图参数：`maps/map.yaml`

### 2.2 导航与控制

- Nav2 参数：`config/nav2_params.yaml`
- 导航与任务组合启动：`launch/nav2_mission.launch.py`

### 2.3 任务执行（课程核心）

- 多站点任务：`autonomous_tb/campus_delivery_node.py`
- 传感器让行：`autonomous_tb/yield_controller_node.py`
- 任务参数：`config/campus_delivery_params.yaml`
- 任务层独立启动：`launch/campus_delivery.launch.py`

### 2.4 数据采集与结果输出

- 数据采集节点：`autonomous_tb/data_collection_node.py`
- 到站拍照输出：`~/ros2_proj_data/photos/`
- 任务日志输出：`~/ros2_proj_data/logs/delivery_log.csv`

## 3. 运行时序（推荐）

1) 启动 Gazebo（课程要求）
2) 启动 Nav2 + 任务层（本项目）
3) RViz 设置初始位姿（2D Pose Estimate）
4) 自动执行多站点任务，生成图片和日志

## 4. 与课程评分项对应

- 仿真环境：Gazebo + TurtleBot3
- 传感器行为：`/scan` 驱动让行暂停/恢复
- 自定义节点开发：任务节点 + 让行节点
- 导航算法：Nav2（全局+局部规划）
- 使用地图：地图加载导航执行
- 地图绘制：探索模式 + map_saver

## 5. 架构原则

- 将“仿真启动”与“任务逻辑”解耦，降低单一 launch 失败影响。
- 任务层可独立复用到真实机器人（替换底层驱动和地图源）。
- 输出统一落盘，便于报告、复现实验和视频取证。
